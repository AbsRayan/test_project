# LiDAR Simulator

Приложение для визуализации 3D-сцен, генерации синтетических данных ToF-камеры и экспорта наборов кадров и облаков точек для задач компьютерного зрения.

Проект реализован как desktop-приложение на `PyQt6` и `OpenGL`. В текущей версии поддерживаются:

- интерактивный просмотр 3D-сцены;
- загрузка и переключение сцен из каталога `configs/`;
- изменение положения самолёта и параметров камер;
- сохранение карты глубины ToF-камеры;
- офлайн-рендер сцены;
- экспорт пролёта камеры в виде RGB-кадров, depth map и point cloud.

---

## Возможности

### Работа со сценами
Приложение автоматически находит файлы `scene*.json` в каталоге `configs/` и позволяет переключаться между ними через графический интерфейс.

На данный момент в проекте предусмотрены следующие сцены:

- `scene.json` — базовая сцена с площадкой, зданиями и несколькими самолётами;
- `scene_airplane_surface.json` — простая сцена с самолётом на поверхности;
- `scene_sphere_surface.json` — простая сцена со сферой на поверхности.

### Интерактивное управление
Из панели управления можно изменять:

- позицию и вращение самолёта;
- позицию и направление ToF-камеры;
- позицию и направление render-камеры;
- параметры маршрута пролёта камеры;
- каталог сохранения результатов;
- формат point cloud: `PCD` или `LAS`.

Навигация в окне сцены:

- **ЛКМ** — вращение обзорной камеры;
- **ПКМ** — панорамирование;
- **колесо мыши** — зум.

### Генерация выходных данных
Приложение позволяет получать и сохранять:

- **ToF depth map**;
- **raytrace-рендер**;
- **набор данных пролёта камеры**, включающий:
  - RGB-кадры,
  - карты глубины,
  - облака точек.

- [`output_images/`](./output_images) — результаты снимков и экспортов.

## Выходные данные

Во время работы приложение сохраняет результаты в каталог `output_images/`.

Структура выходных данных:

- `output_images/heatmaps/` — карты глубины ToF-камеры;
- `output_images/raytraces/` — результаты офлайн-рендера;
- `output_images/flythroughs/` — данные, полученные при экспорте пролёта камеры.

Для каждого пролёта создаётся отдельная сессия с подкаталогами:

- `renders/` — RGB-кадры;
- `depth_maps/` — карты глубины;
- `point_clouds/` — облака точек в формате `PCD` или `LAS`.

## Режимы пролёта камеры

Экспорт пролёта поддерживает два режима:

### `linear`
Линейный пролёт между начальной и конечной точками с сохранением направления взгляда.

### `orbit`
Облёт вокруг выбранной цели по окружности.

В обоих режимах на каждом шаге сохраняются:

- RGB-рендер;
- depth map;
- point cloud.
